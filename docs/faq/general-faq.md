# 一般常見問題 (FAQ)

## Q: 如何搜尋知識庫？

在知識庫網站的搜尋框中輸入關鍵字即可。系統會同時搜尋文件標題、標題層次與內文。

建議使用**具體的關鍵字**而非完整句子，例如：
- ✅ `JWT token 過期`
- ✅ `database connection pool`
- ❌ `我想要知道如何處理 JWT token 過期的問題`

## Q: 找不到答案怎麼辦？

1. 嘗試不同的關鍵字組合
2. 使用「找不到答案」按鈕建立 Issue
3. AI 會在 5 分鐘內自動回答
4. 若 AI 也無法回答，Issue 會被標記為 `knowledge-gap`，等待維運者補充

## Q: AI 的回答一定正確嗎？

不一定。AI 的回答基於知識庫現有文件，若文件資訊不完整或有誤，AI 的回答也可能不準確。

每個 AI 回答都有**信心指數**：
- 高信心（> 80%）：答案通常可靠
- 中信心（50-80%）：建議進一步確認
- 低信心（< 50%）：可能需要人工確認

若發現回答有誤，請在 Issue 中留言說明，維運者會修正知識庫。

## Q: 如何確認 AI 的回答已被人工驗證？

已驗證的 Issue 會有 `human-verified` 標籤（綠色）。你可以在 GitHub Issues 頁面篩選查看。

## Q: 知識庫多久更新一次？

知識庫是**即時更新**的。任何維運者 push 新文件到 `docs/` 目錄後，搜尋索引會在 2-3 分鐘內自動更新。

## Q: 可以直接在 GitHub 上編輯文件嗎？

可以。在 GitHub 上直接編輯 `docs/` 下的 `.md` 文件並 commit，效果與 clone 後 push 相同。適合快速修正錯字或補充說明。
